# 配信ビットレート計算機 📶

ライブ配信の設定(解像度・fps・コーデック・映像の動き)から、
推奨ビットレート・必要な上り回線速度・録画容量を計算するアプリです。
完全オフライン計算・PWA対応。

## 主な機能

- **配信先の選択**:YouTube Live / Twitch / Vimeo / Zoom / 汎用RTMP
  - 各プラットフォームの上限(Twitch=6Mbps・1080pまで・音声160kbps等)で自動クランプ
  - **Zoomは専用モード**:ビットレート指定ができない仕組みのため、画質ごとの必要上り回線の目安
    (1080p=3.8Mbps送信など公式推奨帯域)と判定を表示
- **推奨ビットレート**:YouTube Live推奨値(H.264)ベース。HEVC(×0.65)/AV1(×0.5)換算、
  映像の動き(セミナー/標準/ライブ・スポーツ)による補正付き
- **必要上り回線速度**:総ビットレート×ヘッドルーム(×2/×2.5/×3)で算出
- **録画・アーカイブ容量**:GB/時間、2時間配信の目安
- **回線から逆引き**:会場で実測した上り速度を入れると、
  「この設定でいけるか」の判定(◯/△/✕)と、解像度×fps全組み合わせの判定表を表示

## 現場での使い方

1. 会場に着いたらスピードテスト(fast.comなど)で**上り**を実測
2. 実測値を「回線から逆引き」に入力
3. ◯のついた設定の中から選んでエンコーダーに設定

> 上り回線は時間帯やイベント本番中の来場者Wi-Fi利用で変動します。
> 本番はヘッドルーム×2.5以上と、別キャリアのバックアップ回線推奨。
